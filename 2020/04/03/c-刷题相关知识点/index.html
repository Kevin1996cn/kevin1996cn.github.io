6.关于Int32位反转之后溢出的问题

if(res>INT_MAX/10||(res==INT_MAX/10&&t>7)) return 0;
if(res<INT_MIN/10||(res==INT_MIN/10&&t<-8)) return 0;

2^31-1=2147483647,-2^31=-2147483648

第七题：

class Solution {
public:
    int reverse(int x) {
        int res=0;
        while(x!=0)
        {
                int t=x%10;
                x=x/10;
                if (res > INT_MAX / 10 || res < INT_MIN / 10) //这句可以抵上边两句
                return 0;
                res=res*10+t;

        }
        return res;
    }

官方给出的溢出判断是因为MAX_VALUE : 2147483647 MIN_VALUE : -2147483648 他要判断最后一位不要大于 7 和 不要小于-8.

其实完全没必要验证大于7小于八。首位是7的时候最后一次数才是7，然而int类型最大的数首位是2。

因为你传递的参数，是不能保存首位大于7的数。

int类型的范围是 -2^31——2^31-1，即-2147483648——2147483647，代码中的if中有(rev == INT_MAX / 10 && pop > 7)和(rev == INT_MIN / 10 && pop < -8)判断，看似完全顾及到各种情况，但是仔细一想，会发现根本就不会有符合这个条件的32位int数值，假如符合，也应该是int范围最大值反转过来，然后最高位大于7，首先int范围最大值反转过来就没法当作int数值传递到这个方法的形参中，在编译期就会报错：Integer number too large，所以如果方法的形参不是int类型的话，可以加上这些判断，但是如果方法的形参是int的话，完全不用考虑这些判断，因为在传递这个值的时候，编译期就报错了

9.素数：厄拉多塞筛法 （用来求大量的素数）

具体操作：先将 2~n 的各个数放入表中，然后在2的上面画一个圆圈，然后划去2的其他倍数；第一个既未画圈又没有被划去的数是3，将它画圈，再划去3的其他倍数；现在既未画圈又没有被划去的第一个数 是5，将它画圈，并划去5的其他倍数……依次类推，一直到所有小于或等于 n 的各数都画了圈或划去为止。这时，表中画了圈的以及未划去的那些数正好就是小于 n 的素数。

vector<bool> num(n,true);
int res=0;
for(int i=2;i<=sqrt(n);i++)
{
    if(num[i])
    {
         int k=2;
         while(k*i<n)
         {
             num[k*i]=false;
             k++;
         }
    }
}

12.全排列第46题

class Solution {
public:
    vector<vector<int>> permute(vector<int>& nums) {
    vector<vector<int>> result;
		sort(nums.begin(), nums.end());
		result.push_back(nums);
		while (next_permutation(nums.begin(), nums.end()))
		{
			result.push_back(nums);
		}
		return result;。
};
